## Shared-file rules

- `compare.js` — multiple agents may read; only `accuracy-lift-worst-site` may write, and only outside line 167.
- `package.json` — no agent adds dependencies without explicit approval. Comment in PR if you need one.
- `tests/` — namespaced subfolders per agent (`tests/perf/`, `tests/io-failsafes/`, `tests/algo/`). No edits to other agents' test folders.
- `memory/` files — each agent writes only its own named output file.

## Regression gate (all agents)

Before merging:
1. Run all four site regressions (SPK02, VO201, RIN01, EWS01) plus BVI01 once it has a regression file.
2. No site may regress by more than 2pp hall accuracy.
3. If a regression appears, revert and document in your memory output file.

## Stop-and-ask triggers

- Refactor > 20 lines outside your lane → stop, ask.
- New runtime dependency → stop, ask.
- Touching `compare.js:167` for any reason → stop, ask.
- Algorithm change that regresses any other site > 2pp → revert, then ask.
